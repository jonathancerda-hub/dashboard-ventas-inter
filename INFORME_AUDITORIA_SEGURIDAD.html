🔍 Vulnerabilidades Detectadas y Recomendaciones

1. Content Security Policy con Directivas Inseguras
MEDIA
🏷️ Archivo app.py | Líneas: 161-171

El CSP configurado incluye 'unsafe-inline' y 'unsafe-eval' que debilitan las protecciones contra XSS al permitir scripts inline y eval().

csp_policy = ( "default-src 'self'; " "script-src 'self' 'unsafe-inline' 'unsafe-eval' https://cdn.jsdelivr.net; " # ^^^^^^^^^^^^^^^ INSEGURO "style-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net; " )
ISO/IEC 27001:2022 - Control A.14.2.9: Los servicios web deben implementar mecanismos de control de contenido para prevenir la inyección de código malicioso.

✅ Código Corregido Recomendado

# Opción 1: Usar nonces para scripts inline (recomendado) import secrets @app.before_request def set_csp_nonce(): return secrets.token_urlsafe(16) @app.after_request def add_security_headers(response): nonce = g.get('csp_nonce', '') response.headers['Content-Security-Policy'] = ( f"default-src 'self'; " f"script-src 'self' 'nonce-{nonce}' https://cdn.jsdelivr.net; " f"style-src 'self' 'nonce-{nonce}' https://cdn.jsdelivr.net; " ) return response # En templates: <script nonce="{{ g.csp_nonce }}">...</script>
2. Rate Limiting Básico (Solo por IP)
BAJA
🏷️ Archivo app.py | Línea: 108

El rate limiting actual se basa únicamente en dirección IP, lo que puede ser evadido con proxies o causar falsos positivos en redes corporativas con NAT.

limiter = Limiter( app=app, key_func=get_remote_address, # ⚠️ Solo IP default_limits=["200 per day", "50 per hour"] )

✅ Código Corregido Recomendado

# Implementar rate limiting híbrido (IP + Usuario) def rate_limit_key(): if 'user' in session: return f"user:{session['user']['email']}" return f"ip:{get_remote_address()}" limiter = Limiter( app=app, key_func=rate_limit_key, # ✅ Híbrido IP + Usuario default_limits=["200 per day", "50 per hour"], storage_uri="memory://" ) # Límites más estrictos para endpoints sensibles @app.route('/export') @limiter.limit("10 per hour") # ✅ Usuario autenticado: max 10/hora def export_data(): pass
3. Falta de Rotación Automática de Secrets
MEDIA
🏷️ Archivos .env, app.py, odoo_manager.py

Las claves secretas (SECRET_KEY, GOOGLE_CLIENT_SECRET, ODOO_PASSWORD) no tienen mecanismo de rotación automática. Si se comprometen, no hay proceso para regenerarlas de forma segura.

# .env (sin rotación) SECRET_KEY=abc123... # ⚠️ Sin fecha de expiración GOOGLE_CLIENT_SECRET=xyz789... ODOO_PASSWORD=password123 # ⚠️ Nunca cambia
ISO/IEC 27001:2022 - Control A.8.24: Las claves criptográficas deben estar sujetas a un ciclo de vida de gestión, incluyendo generación, distribución, almacenamiento, rotación y destrucción segura.

✅ Código Corregido Recomendado

# scripts/check_secret_age.py - Script para verificar rotación import os from datetime import datetime, timedelta from pathlib import Path def check_env_file_age(): env_path = Path('.env') if not env_path.exists(): return modified_time = datetime.fromtimestamp(env_path.stat().st_mtime) days_old = (datetime.now() - modified_time).days if days_old >= 90: print(f"⚠️ ALERTA: .env tiene {days_old} días sin modificar") print("🔄 Considera rotar SECRET_KEY y otras credenciales") # Ejecutar con cron job cada semana: # 0 9 * * MON python scripts/check_secret_age.py
4. Whitelist de Usuarios en Texto Plano
BAJA
🏷️ Archivo whitelist.txt (o variableRENV WHITELIST_EMAILS)

La lista de usuarios autorizados se almacena en texto plano. Si un atacante obtiene acceso, conocerá todos los usuarios autorizados, facilitando ataques de phishing dirigido.

# whitelist.txt jonathan.cerda@agrovetmarket.com maria.lopez@agrovetmarket.com # ⚠️ Emails visibles en texto plano

✅ Código Corregido Recomendado

# Almacenar hashes SHA-256 en lugar de emails import hashlib def hash_email(email: str) -> str: return hashlib.sha256(email.lower().encode()).hexdigest() # whitelist_hashes.txt # a1b2c3d4e5f6... (hash de jonathan.cerda@agrovetmarket.com) # x9y8z7w6v5u4... (hash de maria.lopez@agrovetmarket.com) def is_user_authorized(email: str) -> bool: email_hash = hash_email(email) with open('whitelist_hashes.txt') as f: return email_hash in {line.strip() for line in f}
