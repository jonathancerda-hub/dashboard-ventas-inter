2️⃣ Entradas y Salidas

Estado: ❌ NO CUMPLE - CRÍTICO PARA ISO

📋 Criterio: Validación de Entradas

❌ Violaciones Identificadas

🔴 Falta validación de tipos y rangos
# VULNERABLE: app.py línea 527
año = int(request.args.get('año', datetime.now().year))  # ❌ Sin try-catch
# Si 'año' = "'; DROP TABLE metas--", causa 500 error

# VULNERABLE: app.py línea 390
cliente_id = int(request.args.get('cliente_id'))  # ❌ Sin validación
# Permite valores negativos, fuera de rango, None
Impacto ISO:
  • ❌ NO CUMPLE ISO 27034 A.14.2.1 (Validación de datos de entrada)
  • ❌ NO CUMPLE CWE-20 (Improper Input Validation)
  • ❌ NO CUMPLE PCI-DSS 6.5.1 (Injection Flaws)

✅ Solución Requerida: Pydantic

from pydantic import BaseModel, Field, validator

class DashboardQueryParams(BaseModel):
    año: int = Field(
        default_factory=lambda: datetime.now().year,
        ge=2020,  # ✅ Greater or equal
        le=2030,  # ✅ Less or equal
    )
    
    linea: Optional[str] = Field(
        default='',
        max_length=50,
        regex=r'^[A-Z0-9_-]*$',  # ✅ Solo alfanuméricos
    )
    
    cliente_id: Optional[int] = Field(default=None, ge=1)

# Uso en rutas
try:
    params = DashboardQueryParams(**request.args.to_dict())
except ValidationError as e:
    flash('Parámetros inválidos', 'error')
    return redirect(url_for('dashboard'))

📋 Criterio: Sanitización XSS

Estado: ⚠️ CUMPLIMIENTO PARCIAL (60%)

✅ Cumple

Jinja2 auto-escape activo por defecto

❌ Riesgos

🟠 CDN sin Subresource Integrity (SRI)
<!-- VULNERABLE -->
<script src="https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js"></script>
<!-- ❌ Sin integrity hash - si CDN comprometido = XSS -->

<!-- CORRECTO -->
<script 
    src="https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js"
    integrity="sha384-[HASH]"
    crossorigin="anonymous"></script>

3️⃣ Autenticación y Autorización

📋 Criterio: Privilegio Mínimo (RBAC)

Estado: ❌ NO CUMPLE - CRÍTICO PARA ISO

❌ Problema: Sin Role-Based Access Control

Sistema actual: Solo verifica si usuario está en allowed_users.json

Problemas ISO:

  • ❌ Todos los usuarios autorizados tienen acceso completo
  • ❌ Sin diferenciación admin vs. usuario estándar
  • ❌ Sin permisos a nivel de recurso
  • ❌ Violación de ISO 27001 A.9.2.3 (Gestión de privilegios de acceso)

📋 Criterio: Gestión de Sesiones

Estado: ⚠️ CUMPLIMIENTO PARCIAL (75%)

✅ Cumple

  • SESSION_COOKIE_SECURE = True (Solo HTTPS)
  • SESSION_COOKIE_HTTPONLY = True (No JS access)
  • Expiración de sesión: 15 minutos
  • Validación de state token (CSRF protection)

  • Regenerar session ID después de login (prevenir session fixation)

6️⃣ Control de Entorno y Repositorio

📋 Criterio: SCA - Vulnerabilidades en Dependencias

Estado: ⚠️ NO VERIFICADO - REQUIERE AUDIT

❌ Falta de Auditoría Continua

Archivo requirements.txt no auditado regularmente (15+ dependencias)

Riesgos:

  • ❌ Dependencias desactualizadas con CVEs conocidos
  • ❌ Sin verificación automática en CI/CD
  • ❌ Sin política de actualización

✅ Solución: Pipeline de Seguridad

1. Auditoría Local
pip install pip-audit safety
pip-audit
safety check --json
2. Pre-commit Hooks
pip install pre-commit
pre-commit install
# Cada commit ejecuta auditoría
3. CI/CD Pipeline
- name: Run pip-audit
  run: pip-audit
  
- name: Fail on HIGH/CRITICAL
  run: exit 1 if HIGH > 0
4. Dependabot
# .github/dependabot.yml
updates:
  - package-ecosystem: "pip"
    schedule: weekly

📋 Criterio: SAST - Análisis Estático Pre-Commit

Estado: ❌ NO CUMPLE - CRÍTICO PARA ISO

❌ Estado Actual

  • ❌ Sin herramientas SAST configuradas
  • ❌ Sin análisis estático en CI/CD
  • ❌ Sin quality gate antes de merge

✅ Solución: SAST Pipeline (Bandit + SonarCloud)

# Bandit (SAST para Python)
bandit -r . -f json -o bandit-report.json

# Quality Gate
if [ "$BANDIT_HIGH" -gt 0 ]; then
  echo "❌ Quality Gate FAILED"
  exit 1
fi

Umbrales de Calidad ISO:

  • Severidad BLOCKER: 0 issues permitidos
  • Severidad HIGH/CRITICAL: Bloquea merge en PR
  • Cobertura de tests: > 80% (target)
