📊 1. Introducción y Propósito

El gráfico "Avance de Facturación por Producto" es una herramienta visual que combina dos métricas críticas: el porcentaje de facturación de cada producto respecto a su pedido total, y el tiempo transcurrido desde la confirmación del pedido. Esta combinación permite identificar rápidamente productos en riesgo que requieren atención inmediata.

🎯 Objetivos Principales

🚨 Detección Temprana

Identificar pedidos que están cerca de vencer el plazo de 150 días sin haber completado su facturación.

📈 Priorización

Permitir al equipo comercial priorizar productos críticos según urgencia temporal y avance comercial.

👁️ Visibilidad

Proporcionar una vista consolidada del estado de todos los productos pendientes de facturación.

📊 Análisis

Facilitar el análisis por línea comercial, categoría y estado para toma de decisiones estratégicas.

📍 Ubicación en el Dashboard

ℹ️

Visibilidad del Gráfico

Este gráfico aparece solo en la vista general del dashboard (cuando no hay ningún cliente seleccionado). Al hacer clic en un cliente específico, el gráfico se oculta para dar paso a vistas detalladas del cliente.

⏱️ 2. Política de Despacho de 150 Días

📋 Definición de la Política

La empresa tiene establecida una política de despacho que estipula que todos los pedidos deben ser completamente facturados y despachados en un plazo máximo de 150 días desde la fecha de entrega comprometida (commitment_date) del pedido en los estados: credit, sale o done.

📅 Línea de Tiempo del Pedido

📝

Día 0: Fecha de Entrega Comprometida

El pedido entra en estado credit, sale o done. Se registra la fecha de entrega comprometida al cliente (campo commitment_date en Odoo). A partir de esta fecha comienzan a contar los 150 días.

📦

Días 1-90: Período Óptimo

Timeframe ideal para comenzar la facturación. Productos en este rango con buen avance (>80%) se consideran 🟢 Exitosos.

⚠️

Días 91-120: Zona de Alerta

Se requiere atención. Si el avance es menor al 80%, el producto entra en estado 🟡 En Progreso.

🚨

Días 121-150: Crítico

Últimos 30 días del plazo. Productos con avance menor al 80% se marcan como 🔴 Críticos.

❌

Día 151+: Vencido

Pedido fuera de política. Requiere acción inmediata. Indicador 🚨 VENCIDO visible.

🔢 Cálculo de Días

SI Fecha de Entrega > Fecha Actual (entrega en el futuro):
    Días Transcurridos = 0
    Días Restantes = 150

SI Fecha de Entrega ≤ Fecha Actual (entrega en el pasado o hoy):
    Días Transcurridos = Fecha Actual - Fecha de Entrega
    Días Restantes = 150 - Días Transcurridos

// Ejemplo 1: Fecha de entrega futura
Fecha Entrega: 2026-03-31
Fecha Actual: 2026-02-24
Días Transcurridos: 0 días (plazo aún no comienza)
Días Restantes: 150 días

// Ejemplo 2: Fecha de entrega pasada
Fecha Entrega: 2026-01-01
Fecha Actual: 2026-02-24
Días Transcurridos: 54 días
Días Restantes: 96 días

🧮 3. Lógica de Cálculo de Estados

⚡

Lógica Combinada: Tiempo + Avance

El estado de cada producto NO se determina únicamente por su porcentaje de avance, sino por una combinación inteligente entre el tiempo restante y el avance comercial. Esta lógica permite priorizar productos que están cerca de vencer el plazo, incluso si tienen un avance aceptable.

📐 Fórmula de Estados

IF (días_restantes === null OR porcentaje >= 100):
    RETURN "🟢 Exitoso"

ELSE IF ((días_restantes < 30 AND porcentaje < 80) OR (días_restantes < 60 AND porcentaje < 40)):
    RETURN "🔴 Crítico"

ELSE IF (días_restantes < 90 OR porcentaje < 80):
    RETURN "🟡 En Progreso"

ELSE:
    RETURN "🟢 Exitoso"

🔍 6. Filtros Disponibles

Controles de Filtrado

📋 1. Línea Comercial

Opciones:

  • Todas las líneas (predeterminado)
  • PETMEDICA
  • AGROVET
  • AVINET
  • Otras líneas disponibles

Uso: Permite analizar el desempeño por unidad de negocio. Útil para gerentes de línea que quieren ver solo sus productos.

🏷️ 2. Categoría de Producto

Opciones:

  • Todas las categorías (predeterminado)
  • Categorías extraídas automáticamente de Odoo

Uso: Segmentación adicional para análisis granular. Por ejemplo, ver solo "Antiparasitarios" o "Suplementos".

🚦 3. Estado (Tiempo + Avance Combinado)

Opciones:

  • Todos los estados (predeterminado)
  • 🔴 Crítico: Productos que requieren atención inmediata
  • 🟡 En Progreso: Productos que requieren seguimiento
  • 🟢 Exitoso: Productos en buen estado

Uso: CRÍTICO para priorización. El gerente comercial puede filtrar solo productos críticos al inicio del día para tomar acción inmediata.

💡 Estrategias de Filtrado Recomendadas

🚒 Modo "Apagar Incendios"

Filtros: Estado = Crítico
Objetivo: Ver solo productos que requieren acción inmediata este mismo día.

📊 Análisis por Línea

Filtros: Línea Comercial = PETMEDICA, Estado = Todos
Objetivo: Evaluar el desempeño general de una línea de negocio específica.

🎯 Seguimiento Semanal

Filtros: Estado = En Progreso
Objetivo: Productos que aún no son críticos pero necesitan monitoreo regular.

✅ Validación de Éxitos

Filtros: Estado = Exitoso
Objetivo: Identificar buenas prácticas y productos que están cumpliendo plazos.

📚 7. Guía de Interpretación

❓ Preguntas Frecuentes

❓

¿Por qué un producto con 70% de avance aparece como Crítico?

El estado NO se determina solo por el porcentaje. Si ese producto tiene menos de 30 días restantes, aunque tenga 70% de avance (que es menor al 80%), será marcado como Crítico. La lógica prioriza el factor temporal cuando el plazo se está agotando.

❓

¿Un producto con 100% facturado puede ser Crítico?

No. Los productos con 100% de avance siempre se marcan como Exitosos (🟢), sin importar cuántos días hayan transcurrido. Si ya está completamente facturado, se considera cumplido.

❓

¿Qué significa el emoji 🚨 al lado del porcentaje?

El emoji 🚨 indica que el producto está VENCIDO (más de 150 días transcurridos sin completar facturación). Es el nivel más alto de urgencia y requiere escalamiento inmediato.

❓

¿Algunos productos no tienen indicador de tiempo?

Sí. Solo los productos que tienen pedidos pendientes con fecha registrada mostrarán información de tiempo. Los productos 100% facturados no necesitan mostrar días restantes.

🎓 Mejores Prácticas de Uso

🌅 Revisión Matutina

Cada mañana, filtrar por 🔴 Crítico y crear lista de acciones del día. Asignar responsables para cada producto crítico.

📅 Reunión Semanal

Revisar productos 🟡 En Progreso para identificar tendencias y prevenir que pasen a críticos. Ajustar estrategias.

📈 Análisis Mensual

Comparar cantidad de productos por estado mes a mes. ¿Están disminuyendo los críticos? ¿Mejora el avance promedio?

🏆 Reconocimiento

Identificar vendedores o líneas con mayor proporción de productos 🟢 Exitosos para compartir buenas prácticas.

⚙️ 8. Detalles Técnicos de Implementación

🗄️ Origen de Datos

💾

Fuentes de Datos en Odoo

Facturado: Suma de amount_currency de account.move.line (facturas confirmadas)
Pendiente: Suma de qty_to_invoice * price_unit de sale.order.line (estados: credit, sale, done)
Fecha: Campo commitment_date del sale.order asociado (fecha de entrega comprometida)
Código Producto: Campo default_code del product.product (usado como key única)

🔑 Agregación por Código de Producto

// Backend (app.py) - Agregación de datos
for codigo in all_product_codes:
    facturado = facturado_by_product[codigo]
    pendiente = pendiente_by_product[codigo]
    fecha_pedido = fecha_pedido_by_product[codigo] // Fecha más antigua
    
    dias_transcurridos = (datetime.now() - fecha_pedido).days
    dias_restantes = 150 - dias_transcurridos
    
    products_chart_data.append({
        'codigo': codigo,
        'facturado': facturado,
        'pendiente': pendiente,
        'porcentaje': (facturado / total * 100),
        'dias_transcurridos': dias_transcurridos,
        'dias_restantes': dias_restantes
    })

🎨 Frontend: Cálculo de Estado y Renderizado

// Frontend (JavaScript) - Función calcularEstadoProducto
function calcularEstadoProducto(producto) {
    const pct = producto.porcentaje || 0;
    const diasRestantes = producto.dias_restantes;
    
    // Si no hay fecha o ya está 100% facturado
    if (diasRestantes === null || pct >= 100) {
        return 'exitoso';
    }
    
    // Crítico
    if ((diasRestantes < 30 && pct < 80) || (diasRestantes < 60 && pct < 40)) {
        return 'critico';
    }
    
    // En progreso
    if (diasRestantes < 90 || pct < 80) {
        return 'en_progreso';
    }
    
    return 'exitoso';
}

📊 Paginación y Performance

📄 20 Productos por Página

El gráfico muestra 20 productos por página para mantener legibilidad. La paginación es dinámica y se ajusta al filtrado.

⚡ Agregación en Backend

El cálculo de días se hace en el servidor (Python) para reducir carga en el navegador. Los datos llegan pre-calculados al frontend.

🔄 Filtrado en Tiempo Real

Los filtros operan sobre los datos ya cargados en memoria del navegador, proporcionando respuesta instantánea sin recargar la página.

📏 Altura Dinámica

El gráfico ajusta su altura automáticamente según la cantidad de productos mostrados (30px por producto + margen).

🔄 Actualización de Datos

🔄

Frecuencia de Actualización

Los datos se actualizan cada vez que se recarga la página del dashboard. Para ver información más reciente, usar F5 o Ctrl+R (Windows/Linux) o Cmd+R (Mac). Los días restantes se calculan en tiempo real basados en la fecha actual del servidor.

🎯 Conclusión

El gráfico "Avance de Facturación por Producto" es una herramienta poderosa que combina métricas temporales y comerciales para proporcionar una visión integral del estado de los pedidos. Al utilizar la lógica combinada de tiempo + avance, permite identificar y priorizar productos en riesgo de manera mucho más efectiva que sistemas basados únicamente en porcentajes.

Recuerda: El objetivo no es solo ver el avance, sino anticipar problemas y tomar acción proactiva antes de que los pedidos venzan el plazo de 150 días. Usa los filtros estratégicamente y establece rutinas diarias de revisión para maximizar el impacto de esta herramienta.
